BT Mobile Apps

Rebuilding two native codebases into one.

The challenge

British Telecom's consumer mobile apps for iOS and Android had been built by a previous vendor as two separate native codebases. As platform requirements evolved, maintaining and updating both independently had become expensive and slow. New features required parallel implementation on two stacks, doubling effort and introducing inconsistencies in behaviour and UX across platforms. BT needed to modernize the apps without disrupting millions of active users, and to do so with a team who could work within their established backend architecture.

What we did

People More scoped a migration to a single React Native codebase that would run natively on both platforms. The work involved three parallel tracks: UX redesign for both platforms (addressing years of accrued inconsistency), co-design of a new API layer with BT's backend engineering team to expose the data and services the new app required, and incremental migration of features from both legacy codebases to the unified React Native build.

The phased approach ensured that live app availability was maintained throughout. Features were released incrementally, with the legacy versions remaining in production until the new codebase reached full feature parity.

Delivery milestones

Phase Deliverable
Discovery & audit Codebase assessment, feature inventory, UX baseline review
API design New API layer co-designed with BT backend team; contract defined
UX redesign Unified design system; iOS and Android interaction specifications
Core build Authentication, account management, billing, and support features in React Native
Parity release Full feature parity with legacy apps; parallel live testing
Cutover Legacy apps decommissioned; single React Native codebase in production

Results

Reduction in development cost versus maintaining two separate native codebases
